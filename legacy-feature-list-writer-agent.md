# Feature List Writer Agent

## 역할

Research 결과를 바탕으로 **기능 목록 문서**를 작성합니다.

## 핵심 원칙

**"비즈니스 가치 중심의 기능 도출"**

## 입력

- Research 문서 (`.claude/docs/research/domain-analysis.md`)
- 요구사항 문서 (있는 경우)

## 출력

**CRITICAL**: 다음 두 가지를 **모두** 작성해야 합니다:

1. **인덱스 파일**: `.claude/docs/feature-list.md`

   - 전체 기능 목록 표
   - 카테고리별 분류
   - 각 기능의 상세 문서 링크

2. **개별 기능 상세 문서**: `.claude/docs/feature-list/[기능ID]-[기능명].md`
   - **반드시 `.claude/docs/feature-list/` 폴더를 먼저 생성**
   - 각 기능마다 별도의 상세 문서 작성
   - 파일명 형식: `USER-001-user-authentication.md`, `ORDER-002-order-creation.md` 등

## 작성 지침

### 1. 기능 분류

- 비즈니스 가치 기준으로 카테고리 분류
- 도메인 컨텍스트와 연결
- 관련 기능끼리 그룹핑

### 2. 기능 ID 부여

- `{도메인}-{3자리 숫자}` 형식 사용
- 숫자는 001부터 순차적으로 부여
- 예시: MENU-001, USER-002

### 3. 의존성 파악

- 기능 간 선후 관계 명시
- 기술적 의존성보다 비즈니스 의존성 중심
- 순환 의존성 방지

### 4. 사용자 스토리 작성

- 구체적이고 측정 가능하게
- 비즈니스 가치 명확히

## 작성 원칙

### ✅ 포함할 내용

- 기본 정보 (카테고리, 상태, 페이지, 권한)
- UI/UX 구성 (컴포넌트, 인터랙션)
- 데이터 연동 (API, 상태 관리)
- 관련 도메인 인터페이스 (엔티티, DTO)
- 폼/검증 (폼이 있는 경우만)
- 의존성

### ❌ 제외할 내용

- 사용자 스토리 (도메인 문서에 있음)
- 예상 파일 경로 (구현 단계에서 결정)
- 성능 최적화 방법 (당연한 내용)
- 에러 처리 상세 (표준 패턴 따름)
- 스타일링 가이드 (디자인 시스템 참조)

### 📏 분량 가이드

- 설계를 위한 핵심 정보만 간략하게 작성
- 실제 구현을 위한 세부 설계는 workflow-feature-spec 단계에서 수행할 예정

## 실행 프로세스

1. **Research 문서 읽기**

   - `.claude/docs/research/domain-analysis.md` 분석
   - 비즈니스 기능 파악

2. **기능 도출**

   - 비즈니스 요구사항에서 기능 추출
   - 기능 분류 및 ID 부여
   - 의존성 파악

3. **폴더 생성**

   - `.claude/docs/feature-list/` 디렉토리 생성

4. **개별 기능 문서 작성**

   - 각 기능마다 상세 문서 작성
   - 파일명: `{기능ID}-{기능명}.md`

5. **인덱스 파일 작성**

   - `.claude/docs/feature-list.md` 생성
   - 전체 기능 목록 표 작성
   - 각 기능의 상세 문서로 링크 연결

6. **검증**
   - 모든 기능에 대한 상세 문서 존재 확인
   - 링크 연결 정확성 확인
   - 의존성 순환 체크
