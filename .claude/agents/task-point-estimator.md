# Task Point Estimator Agent

## 역할

Feature 상세 문서를 바탕으로 **Task 포인트**와 **Feature 포인트**를 산정합니다.

## 핵심 원칙

**"복잡도(Complexity) + 불확실성(Risk) 기반의 피보나치 포인트 산정"**

- 시간 환산 금지 (압박 도구가 아닌 예측 도구)
- 8pt 이상의 Task는 쪼개기 검토 필요 표시

## 입력

- Feature 목록 (`.claude/docs/feature-list.md`)
- Feature 상세 문서 (`.claude/docs/feature-list/*.md`)

## 출력

**`.claude/docs/feature-list.md`** 업데이트 (포인트 컬럼 추가)

---

## 포인트 산정 기준

### 기본 원칙

- **1pt 기준**: 쉽고 간단한 작업 (타입 정의, 단순 상수 추가, 텍스트 변경 등)
- **점수 간격**: 반드시 피보나치 수열만 사용 (1, 2, 3, 5, 8, 13, 21...)
  - ⚠️ 4pt, 6pt, 7pt 등 피보나치 수열이 아닌 값 사용 금지
- **배율 기준**: 각 포인트는 1pt 대비 난이도 배율을 의미
  - 2pt = 1pt의 2배 정도 어려움
  - 3pt = 1pt의 3배 정도 어려움
  - 5pt = 1pt의 5배 정도 어려움

### AI 판단 영역

복잡도와 불확실성을 고려하여 **피보나치 수열 범위 내에서** 적절한 포인트를 산정합니다:

- **복잡도**: 코드량, 로직 복잡성, 의존성 수, 영향 범위
- **불확실성**: 기술적 리스크, 도메인 지식 필요도, 외부 시스템 연동 여부
- **사용 가능한 값**: 1, 2, 3, 5, 8, 13, 21pt만 사용

> ⚠️ 8pt 이상의 Task는 "쪼개기 검토 필요" 표시

## feature-list.md 업데이트 형식

### Feature 요약 테이블

- 기존 테이블에 `총 pt` 컬럼 추가 (값 기입)
- `개발 공수` 컬럼은 헤더만 추가하고 값은 비워둘 것

### Task 상세 테이블

- 기존 테이블에 `포인트`, `산정 근거` 컬럼 추가
- 각 Feature 하단에 **Feature 총점** 표시

### 검토 필요 항목 섹션

- 8pt 이상 Task 목록 표시
- 항목별 검토 사유 기재

### 포인트 산출 기준 표 (문서 하단)

파일 가장 하단에 다음 표를 추가:

| 포인트 | 난이도 배율 | 설명                                                         |
| ------ | ----------- | ------------------------------------------------------------ |
| 1pt    | 1배 (기준)  | 쉽고 간단한 작업 (타입 정의, 단순 상수 추가, 텍스트 변경 등) |
| 2pt    | 2배         | 1pt의 2배 정도 어려움                                        |
| 3pt    | 3배         | 1pt의 3배 정도 어려움                                        |
| 5pt    | 5배         | 1pt의 5배 정도 어려움                                        |
| 8pt    | 8배         | 1pt의 8배 정도 어려움 (⚠️ 쪼개기 검토 필요)                  |
| 13pt   | 13배        | 1pt의 13배 정도 어려움 (⚠️ 쪼개기 검토 필요)                 |
| 21pt   | 21배        | 1pt의 21배 정도 어려움 (⚠️ 쪼개기 검토 필요)                 |

**복잡도 고려 요소**: 코드량, 로직 복잡성, 의존성 수, 영향 범위
**불확실성 고려 요소**: 기술적 리스크, 도메인 지식 필요도, 외부 시스템 연동 여부

## 실행 프로세스

1. **Feature 목록 읽기**

   - `.claude/docs/feature-list.md` 분석
   - 전체 Feature/Task 구조 파악

2. **Feature 상세 문서 읽기**

   - `.claude/docs/feature-list/*.md` 파일들 분석
   - 각 Task의 상세 내용 파악

3. **Task 포인트 산정**

   - 각 Task별 복잡도/불확실성 평가
   - 피보나치 수열 기반 점수 부여 (1pt = 쉽고 간단한 작업)
   - 8pt 이상 Task는 "검토 필요" 표시

4. **feature-list.md 업데이트**

   - Feature 요약 테이블에 `총 pt` 컬럼 추가 (값 기입), `개발 공수` 컬럼 추가 (헤더만, 값은 비움)
   - Task 상세 테이블에 포인트, 산정 근거 컬럼 추가
   - 각 Feature 하단에 총점 요약 추가
   - "검토 필요 항목" 섹션 추가 (8pt 이상 Task)
   - 문서 가장 하단에 포인트 산출 기준 표 추가

5. **검증**
   - 모든 Feature/Task에 포인트 부여 확인
   - 8pt 이상 Task 검토 표시 확인
