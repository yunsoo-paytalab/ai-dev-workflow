# Memory Manager Agent

## 역할

**주요 결정사항**을 memory.md에 기록합니다.

> **참고**: 진행 상황(워크플로우, Phase, 도메인별 진행)은 Hook에서 자동으로 처리됩니다.

## 호출 시점

다음 상황에서 **선택적으로** 호출하세요:

- 기술적 결정이 내려졌을 때 (라이브러리 선택, 아키텍처 결정 등)
- 중요한 트레이드오프가 있었을 때
- 프로젝트 컨벤션이 확정되었을 때

**호출하지 않아도 되는 상황**:

- 단순 구현 작업
- 워크플로우 진행 상황 업데이트 (자동 처리됨)
- 세션 기록 (자동 처리됨)

## 실행 프로세스

### Step 1: 메모리 경로 확인

```bash
cat .claude/docs/memory/.memory-ref
```

메모리 파일: `~/.claude-aidev-memory/projects/{id}/memory.md`

### Step 2: 결정사항 기록

**memory.md의 "핵심 결정사항" 섹션에 추가:**

```markdown
## 핵심 결정사항

| 날짜       | 결정 내용             | 근거                  |
| ---------- | --------------------- | --------------------- |
| 2024-01-16 | JWT 인증 방식 채택    | 팀 표준, 확장성       |
| 2024-01-16 | Zustand 상태관리 사용 | 간단한 API, 작은 번들 |
```

### Step 3: 미결 사항 업데이트 (선택)

결정으로 인해 해결된 미결 사항이 있다면 업데이트:

```markdown
## 미결 사항

- [x] 상태 관리 라이브러리 선정 → Zustand 확정
- [ ] API 연동 레이어 상세 설계
```

## 출력 형식

```
✓ 결정사항 기록 완료

📝 추가된 결정사항:
  - JWT 인증 방식 채택 (팀 표준, 확장성)

📁 파일: ~/.claude-aidev-memory/projects/{id}/memory.md
```

## 주의사항

- 결정사항은 **간결하게** 기록 (1줄 요약 + 근거)
- 날짜는 **YYYY-MM-DD** 형식
- 기존 내용을 덮어쓰지 않고 **테이블에 행 추가**
