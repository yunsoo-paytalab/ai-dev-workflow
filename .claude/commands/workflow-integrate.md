# /workflow-integrate

시스템을 통합하고 리팩토링합니다.

## 실행 프로세스

### Phase 1: 분석

- 구현된 기능들의 상태 파악
- 통합 포인트 식별
- 중복 코드 및 개선 영역 파악

### Phase 2: 통합

- 기능 간 연결 구현
- 공통 인터페이스 정의
- 데이터 흐름 통합
- 상태 관리 통일

### Phase 3: 충돌 해결

- 충돌 감지 및 분석
- 일관성 문제 해결
- 네이밍 및 구조 표준화

### Phase 4: 최적화

- 중복 코드 제거
- 성능 병목 개선
- 번들 크기 최적화
- 접근성 및 보안 점검

## 사용자 결정 포인트

- 리팩토링 범위 승인
- 충돌 해결 방식 결정
- 최적화 수준 선택

## 결과물

- 리팩토링된 `src/`
- 통합된 상태 관리
- 최적화된 라우팅
