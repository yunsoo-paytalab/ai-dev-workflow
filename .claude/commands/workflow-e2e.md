# /workflow-e2e

E2E 테스트를 작성하고 실행합니다.

## 실행 프로세스

1. **준비** → **Agent: memory-manager**로 상태 확인
2. **시나리오 설계**
   - 주요 사용자 플로우 식별
   - Critical Path 정의
   - 테스트 시나리오 작성
   - 테스트 우선순위 결정
3. **E2E 테스트 케이스 생성**
   - 사용자 시나리오 기반 테스트 케이스 생성
   - Happy path, 예외 상황 포함
   - 페이지 객체 패턴 적용 제안
4. **E2E 테스트 작성**
   - 테스트 환경 설정 확인
   - 시나리오별 테스트 코드 작성
   - 테스트 데이터 준비
5. **테스트 실행**
   - E2E 테스트 실행
   - 실패 케이스 분석
   - 테스트 안정성 확인
   - 스크린샷 및 로그 수집
6. **커버리지 분석**
7. **저장** → **Agent: memory-manager**로 진행 상황 업데이트

## 사용자 결정 포인트

🔔 **사용자 입력 필요**:

- 테스트 시나리오 검토 및 승인
- Critical Path 선택
- 테스트 환경 설정 확인
