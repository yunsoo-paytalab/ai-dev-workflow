# Danger Zones 문서 작성 가이드

## 핵심 원칙

4개 분석 결과를 종합하여 위험도별로 분류합니다.

## 분류 기준

### 🔴 Critical (하나라도 해당)

- 비즈니스 크리티컬 + 커버리지 0%
- 복잡도 >20
- 결합도 >10
- 결제/인증/핵심 도메인

### 🟠 High Risk

- 커버리지 <30%
- 복잡도 15-20
- 결합도 5-10
- 순환 의존성
- 변경 빈도 >50

### 🟡 Medium Risk

- 커버리지 30-70%
- 복잡도 10-15
- 파일 크기 >500 LOC
- 변경 빈도 20-50

### 🟢 Safe

- 커버리지 >70%
- 복잡도 <10
- 결합도 <5

## 문서 구조

```markdown
# Danger Zones 분석

## 개요

| 등급           | 파일 수 | 즉시 조치 필요 |
| -------------- | ------- | -------------- |
| 🔴 Critical    | 5       | 3              |
| 🟠 High Risk   | 12      | 5              |
| 🟡 Medium Risk | 25      | -              |
| 🟢 Safe        | 100+    | -              |

## 🔴 Critical Zone

> **수정 시 반드시 명시적 승인 필요**

| 파일         | 위험 요인      | 커버리지 | 복잡도 | 컨펌 요구사항             |
| ------------ | -------------- | -------- | ------ | ------------------------- |
| `payment.ts` | 결제 핵심 로직 | 0%       | 25     | 영향 분석 + 테스트 + 승인 |

### 상세: payment.ts

**위험 요인:**

- 결제 핵심 로직 (비즈니스 크리티컬)
- 테스트 커버리지 0%
- 복잡도 25 (>20)

**수정 시 필수 조치:**

1. 영향 범위 분석 필수
2. 최소 3개 테스트 케이스 작성
3. 명시적 사용자 승인 필요

## 🟠 High Risk Zone

> **변경 계획 + 승인 필요**

| 파일              | 위험 요인 | 커버리지 | 복잡도 |
| ----------------- | --------- | -------- | ------ |
| `auth/session.ts` | 인증 로직 | 25%      | 18     |

## 🟡 Medium Risk Zone

> **주의 안내 후 진행**

| 파일              | 위험 요인     | 권장 사항        |
| ----------------- | ------------- | ---------------- |
| `utils/format.ts` | 높은 변경빈도 | 테스트 보강 권장 |

## 🟢 Safe Zone

> **자유롭게 수정 가능**

대부분의 컴포넌트, 유틸리티 파일
```
