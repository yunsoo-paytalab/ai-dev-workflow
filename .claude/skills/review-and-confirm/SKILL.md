# Review & Confirm Skill

사용자 검토 및 승인 프로세스를 표준화하는 스킬입니다.

## 사용 시점

워크플로우에서 사용자 검토가 필요한 모든 Phase에서 사용

## 구현 방법

> **AskUserQuestion Tool 사용**하여 선택지 제시

```
AskUserQuestion({
  questions: [{
    question: "위 검토 항목을 확인했습니다. 다음 단계로 진행할까요?",
    header: "검토",
    options: [
      { label: "✅ 승인", description: "다음 Phase로 진행" },
      { label: "💬 수정 요청", description: "피드백 또는 수정 요청" }
    ],
    multiSelect: false
  }]
})
```

## 표준 선택지

### 기본형 (승인/직접입력)

| 선택         | 처리                              |
| ------------ | --------------------------------- |
| ✅ 승인      | 다음 Phase로 진행                 |
| 💬 직접 입력 | 요청 수행 후 **선택지 다시 표시** |

### 커밋형 (Commit 선택지)

| 선택                | 처리                              |
| ------------------- | --------------------------------- |
| ✅ Commit하고 다음  | git commit 후 진행                |
| ⏭️ Commit 없이 다음 | git add만 하고 진행               |
| 💬 직접 입력        | 요청 수행 후 **선택지 다시 표시** |

### 다중 선택형 (상황별)

| 선택         | 처리                              |
| ------------ | --------------------------------- |
| ✅ 옵션 A    | 옵션 A 실행                       |
| ✅ 옵션 B    | 옵션 B 실행                       |
| 💬 직접 입력 | 요청 수행 후 **선택지 다시 표시** |

## 핵심 규칙

> 🚨 **CRITICAL**
>
> 1. **승인 = 선택지에서 "승인" 옵션 선택한 경우만**
> 2. **그 외 모든 입력(URL, 텍스트, "오케이" 등) → 처리 후 선택지 재표시**
> 3. **다음 Phase에 전제조건 있으면 승인 없이 진행 불가**

## 사용 방법

워크플로우 커맨드에서 다음과 같이 참조:

```markdown
### Phase N: Review & Confirm

review-and-confirm 스킬을 사용하여 사용자 검토를 진행합니다.

검토 항목:

- 항목 1
- 항목 2
```

## 직접 입력 처리 예시

```
사용자: "API 응답 형식을 수정해줘"
     ↓
Claude: 요청 수행 (API 응답 형식 수정)
     ↓
Claude: 결과 공유 ("API 응답 형식을 다음과 같이 수정했습니다...")
     ↓
Claude: **선택지 다시 표시** (✅ 승인 / 💬 직접 입력)
     ↓
사용자: "✅ 승인"
     ↓
Claude: 다음 Phase로 진행
```

## 주의사항

- 직접 입력 시 절대 자동으로 승인 처리하지 말 것
- 사용자가 "좋아", "오케이", "진행해" 등 비공식 승인 표현을 해도 선택지 다시 제공
- 반드시 선택지에서 **명시적으로 선택**해야만 진행
