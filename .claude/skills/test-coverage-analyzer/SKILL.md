# 테스트 커버리지 분석기

## 사용 지침

테스트 커버리지를 분석하고 추가 테스트가 필요한 영역을 식별합니다.

## 분석 프로세스

1. **커버리지 도구 실행**

   ```bash
   # JavaScript/TypeScript
   npm run test:coverage

   # Python
   pytest --cov=. --cov-report=html

   # Go
   go test -cover ./...
   ```

2. **결과 파싱**

   - 커버리지 백분율 추출
   - 커버되지 않은 라인 식별
   - 테스트되지 않은 함수 찾기

3. **리포트 생성**
   - 전체 커버리지 메트릭
   - 파일별 세부 분석
   - 중요한 갭 식별

## 커버리지 목표

- **최소**: 전체 커버리지 70%
- **권장**: 중요 경로 80%
- **이상**: 비즈니스 로직 90%

## 리포트 형식

```markdown
# 테스트 커버리지 리포트

## 요약

- 전체 커버리지: 75.5%
- 파일: 45/50 테스트됨
- 함수: 120/150 테스트됨

## 중요한 갭

1. UserAuthentication.js - 45% 커버리지

   - 누락: 에러 처리 테스트
   - 누락: 엣지 케이스

2. PaymentProcessor.js - 60% 커버리지
   - 누락: 환불 로직 테스트

## 권장사항

- 우선순위 1: 인증 에러 테스트 추가
- 우선순위 2: 결제 엣지 케이스 테스트
- 우선순위 3: 유틸리티 함수 커버리지 증가
```

## 테스트 우선순위 매트릭스

| 커버리지 | 비즈니스 영향 | 조치   |
| -------- | ------------- | ------ |
| 낮음     | 높음          | 즉시   |
| 낮음     | 낮음          | 예약   |
| 높음     | 모든 경우     | 모니터 |
