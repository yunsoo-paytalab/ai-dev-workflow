# Feature Detail 문서 작성 가이드

## 핵심 원칙

**"Feature 목록 기반으로 각 Feature의 상세 문서 작성"**

## 포함할 내용

### Feature 수준

- 기본 정보: 카테고리, 상태, 페이지, 권한
- 개요: 기능의 목적과 범위 (2-3줄)
- 주요 요구사항: 기능 요구사항, 비즈니스 규칙, 제약사항
- 데이터 연동: API 엔드포인트, 상태 관리
- 관련 도메인 인터페이스: 엔티티, DTO
- 관련 유저 시나리오: user-scenarios.md에서 해당 Feature와 연결된 시나리오

### Task 수준

- **구현 요소**: API, 훅/유틸, 엔티티
- **조건 분기**: 분기 케이스 나열
- **상태 복잡도**: 공유 범위, 상태 의존성, 전역 상태 여부
- **핵심 비즈니스 로직**: 복잡한 계산/검증의 함수 시그니처 (있는 경우만)

## 제외할 내용

- UI 컴포넌트 구성, 레이아웃, 인터랙션 설명
- 구체적인 파일 경로 / 디렉토리 구조
- 에러 처리 코드 구현 상세
- 테스트 코드 구현 / 상세 테스트 케이스

## 분량 가이드

- Feature 전체: 필요한만큼 상세히 작성
- 각 Task 섹션: 모든 Task에 대해 각 5-10줄 이내
