# 2-3pt Feature 기준

## 정의

**2pt**: 1pt보다 약간 복잡하지만 명확한 Feature
**3pt**: 일반적인 복잡도의 표준 Feature

## 2pt 해당 예시

### 복수 컴포넌트 조합

- 2-3개 컴포넌트 연동
- 간단한 상태 공유
- 기본 레이아웃 구성

### 간단한 상태 관리

- 로컬 상태 + Context
- 단순 폼 상태
- 기본 에러 처리

### 단순 API 연동

- 단일 API 호출
- 기본 로딩/에러 상태
- 캐싱 없음

## 3pt 해당 예시

### 표준 CRUD + 유효성 검사

- 복잡한 폼 유효성 검사
- 에러 메시지 처리
- 다중 필드 검증

### 중간 복잡도 UI

- 조건부 렌더링
- 기본 인터랙션
- 반응형 처리

### 간단한 비즈니스 로직

- 계산 로직
- 조건 분기
- 데이터 변환

## 2pt vs 3pt 판단 기준

| 요소          | 2pt   | 3pt         |
| ------------- | ----- | ----------- |
| 파일 수       | 3-5개 | 5-7개       |
| 상태 관리     | 로컬  | 로컬 + 공유 |
| 비즈니스 로직 | 단순  | 중간        |
| 외부 의존성   | 없음  | 1-2개       |

## 선택 가이드

- 확실히 1pt보다 복잡 → **2pt**
- 5pt까지는 아니지만 복잡함 → **3pt**
- 애매하면 **3pt** 선택 (과소평가 방지)
