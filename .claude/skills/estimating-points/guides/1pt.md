# 1pt Feature 기준

## 정의

**1pt = 최소 단위 Feature**

단일 목적의 간단한 Feature로, 복잡한 로직이나 외부 의존성이 없음.

## 해당 예시

### 타입/Enum 정의

- 단순 인터페이스 정의
- Enum 타입 정의
- 공용 타입 정의

### 공통 유틸리티

- 단순 포맷팅 함수
- 유효성 검사 헬퍼
- 상수 정의

### 단순 조회 화면

- 정적 정보 표시
- 단순 리스트 조회
- 읽기 전용 상세 화면

### 기본 CRUD

- 단일 엔티티 생성/조회/수정/삭제
- 복잡한 비즈니스 로직 없음
- 단순 폼 + API 호출

## 특징

- 명확한 요구사항
- 기존 패턴 재사용
- 외부 의존성 없음
- 단일 파일 또는 2-3개 파일 수정

## 판단 기준

다음 질문에 모두 "예"라면 1pt:

1. 기존 패턴을 그대로 따라할 수 있는가?
2. 복잡한 상태 관리가 필요 없는가?
3. 외부 시스템 연동이 없는가?
4. 비즈니스 로직이 단순한가?
