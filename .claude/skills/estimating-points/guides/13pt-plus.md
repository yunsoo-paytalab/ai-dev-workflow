# 13pt+ Feature 기준

## 정의

**13pt 이상 = 분할 검토 필요**

Feature가 너무 크거나 복잡하여 단일 Feature로 관리하기 어려움.

## 13pt 해당 예시

### 대규모 시스템 통합

- 5개 이상 API 연동
- 복잡한 데이터 동기화
- 다중 시스템 트랜잭션

### 핵심 플랫폼 기능

- 결제 시스템 전체
- 인증/인가 시스템 전체
- 알림 시스템 전체

### 높은 불확실성

- 완전히 새로운 도메인
- 기술적 PoC 필요
- 요구사항 불명확

## 21pt 해당 예시

- 아키텍처 수준 변경
- 플랫폼 마이그레이션
- 완전히 새로운 제품 라인

## 분할 검토 프로세스

### Step 1: 분할 가능성 평가

```
이 Feature를 독립적으로 배포 가능한 단위로 나눌 수 있는가?
```

### Step 2: 분할 기준

- **도메인 경계**: 다른 도메인으로 분리
- **기능 단계**: Phase 1, Phase 2로 분리
- **레이어 분리**: API, UI, 로직 분리
- **사용자 여정**: 주요 흐름 vs 부가 흐름

### Step 3: 분할 예시

#### Before (13pt)

```
결제 시스템 구현
```

#### After (분할)

```
- 결제 수단 등록 (3pt)
- 결제 요청 처리 (5pt)
- 결제 결과 처리 (3pt)
- 결제 내역 조회 (2pt)
```

## 분할 불가 시

일부 Feature는 분할이 어려울 수 있음:

1. **명시적 표시**: "검토 필요 항목" 섹션에 추가
2. **사유 기재**: 왜 분할이 어려운지 설명
3. **리스크 문서화**: 예상되는 리스크 기재

## 검토 필요 항목 형식

```markdown
## 검토 필요 항목

### AUTH-005: 소셜 로그인 통합 (13pt)

- **사유**: 3개 이상 OAuth 제공자 연동 필요
- **분할 가능성**: 제공자별로 분할 검토
- **리스크**: 각 제공자 API 변경에 취약
```
