# 5-8pt Feature 기준

## 정의

**5pt**: 복잡한 Feature, 여러 시스템 연동
**8pt**: 매우 복잡한 Feature, 높은 불확실성

## 5pt 해당 예시

### 복잡한 상태 관리

- 전역 상태 관리 필요
- 여러 컴포넌트 간 상태 동기화
- 낙관적 업데이트

### 다중 API 연동

- 2-3개 API 조합
- 데이터 캐싱
- 에러 복구 로직

### 복잡한 UI/UX

- 드래그 앤 드롭
- 무한 스크롤
- 복잡한 폼 위저드

### 외부 서비스 연동

- 결제 API (간단)
- 소셜 로그인
- 파일 업로드

## 8pt 해당 예시

### 핵심 비즈니스 로직

- 복잡한 계산 로직
- 트랜잭션 처리
- 비즈니스 규칙 엔진

### 시스템 통합

- 3개 이상 API 연동
- 실시간 동기화
- 복잡한 에러 처리

### 고급 기능

- 실시간 협업
- 복잡한 권한 관리
- 워크플로우 엔진

### 높은 불확실성

- 새로운 기술 도입
- 명확하지 않은 요구사항
- 외부 시스템 의존

## 5pt vs 8pt 판단 기준

| 요소        | 5pt    | 8pt       |
| ----------- | ------ | --------- |
| 파일 수     | 7-10개 | 10-15개   |
| API 연동    | 2-3개  | 3개 이상  |
| 상태 복잡도 | 높음   | 매우 높음 |
| 불확실성    | 중간   | 높음      |
| 도메인 지식 | 필요   | 깊이 필요 |

## 선택 가이드

- 복잡하지만 기술적으로 명확 → **5pt**
- 불확실성이 높거나 시스템 통합 → **8pt**
- 8pt가 부담스러우면 **분할 검토**
