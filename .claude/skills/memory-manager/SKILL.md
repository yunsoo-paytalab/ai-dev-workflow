# 메모리 관리자

## 사용 지침

지속적인 메모리 파일을 사용하여 프로젝트 워크플로우 진행 상황을 추적하고 관리합니다.

## 메모리 파일 위치

`docs/workflows/memory.md`

## 작업

1. **(필요 시) 메모리 초기화**

   - `docs/workflows/memory-template.md`에서 새 메모리 파일 생성
   - 초기 프로젝트 상태 설정
   - 예시는 [examples.md](examples.md) 참조

2. **진행 상황 업데이트**

   - 작업 완료 표시 (체크박스 체크)
   - 기능 진행 테이블 상태 업데이트
   - 도메인 목록 및 페이지 구조 업데이트

3. **상태 읽기**
   - 현재 진행 상황 로드
   - 다음 단계 식별
   - 의존성 확인

## 메모리 구조

메모리 파일 템플릿: `docs/workflows/memory-template.md`
구체적인 예시: [examples.md](examples.md)

### 주요 섹션

1. **도메인 목록**: 프로젝트의 도메인 구조
2. **페이지 구조**: URL 경로와 페이지 매핑
3. **전체 진행 상황 체크리스트**: 워크플로우 단계별 체크리스트
4. **기능 진행 상황**: 기능별 상세 진행 상태 테이블

## 업데이트 규칙

- 항상 기존 내용 보존
- 주요 업데이트에 타임스탬프 추가
- 결정 이력 유지
- 작업 의존성 유지
- 예시 참조: [examples.md](examples.md)
