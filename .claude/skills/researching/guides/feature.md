# Feature Research 전략 가이드

## 핵심 원칙

**"요구사항 분석 + 코드베이스 탐색, 설계는 planning에서"**

- 기능의 목적과 사용자 시나리오 명확화
- 비즈니스 규칙 정의
- 기존 코드베이스에서 재사용/참고할 부분 파악
- 데이터 연동 요구사항 정리
- 설계나 구현 계획은 작성하지 않음

## 주요 작업

### 1. 기능 개요 파악

- 기능의 목적과 해결하려는 문제 정의
- 핵심 가치 제안 명확화

### 2. 사용자 시나리오 도출

- 실제 사용 흐름 시나리오 작성
- 주요 케이스 및 엣지 케이스 식별

### 3. 비즈니스 규칙 정의

- 기능이 따라야 할 규칙과 제약사항
- 검증 조건, 권한, 제한사항 정리

### 4. 코드베이스 분석

- 관련 기술 스택 및 의존성 확인
- 해당 영역의 패턴과 컨벤션 분석
- 유사 기능 구현 사례 및 재사용 가능 컴포넌트 찾기
- 기술적 제약사항 파악

**코드 탐색 최적화 (토큰 절감):**

- **Grep 우선 사용** - 전체 파일 읽기 전에 관련 코드 패턴 검색
- **선택적 읽기** - 500줄 이상 파일은 `offset`과 `limit` 파라미터 활용
- **관련 파일 우선** - Feature 상세 문서에 언급된 파일 우선 탐색
- **중복 읽기 방지** - 같은 세션 내 이미 읽은 파일 추적
- **탐색 결과 기록** - section 1.4 "related files"에 모든 탐색 파일 기록 (planning-agent 재사용)

### 5. 데이터 연동 요구사항

- 필요한 API 엔드포인트 식별
- 상태 관리 필요 항목 정리
- 외부 서비스 연동 사항 확인

## 포함하지 말아야 할 내용

1. **상세 코드 분석** - 코드 내용, 구현 상세, 함수 시그니처 등
2. **기술 설계** - Before/After 코드, 인터페이스 정의, API 스펙 등
3. **Implementation Plan** - planning-agent가 별도 파일로 작성
4. **일반적인 패턴 설명** - React 기본 패턴 등 당연한 내용 생략

## 실행 프로세스

```
1. Feature 상세 문서 읽기 (.claude/docs/feature-list/[ID]-*.md)
2. 요구사항 기반 분석 수행
3. 코드베이스 효율적 탐색
4. 분석 결과 문서화 → templates/feature-research.md 참조
```

## 출력

- 파일: `.claude/docs/research/[Feature ID]-research.md`
- 문서 템플릿: `researching` skill의 `templates/feature-research.md` 참조
